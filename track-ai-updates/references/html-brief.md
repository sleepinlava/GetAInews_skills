# HTML 简报设计说明

本 skill 默认把结果整理为中文 HTML 简报，而不是普通文本列表。

## 页面目标

让用户打开页面后，在 10 秒内看清：
- 最近 AI 领域最重要的 3-5 件事
- 每件事来自哪里
- 哪些值得继续跟踪

## 推荐页面结构

### 1. Hero

包含：
- 简报标题
- 时间范围
- 一句话摘要
- 来源数量或平台分布

### 2. Key Signals

展示 3-5 条高优先级动态。
可采用大号标题配短说明，不要做成拥挤的统计卡片墙。

### 3. Timeline

按时间顺序列出主要更新。
每个条目至少包含标题、平台、日期、说明、链接。

### 4. Platform Sections

按平台拆分：
- X
- Hugging Face
- 官方博客
- GitHub

### 5. Watchlist

列出后续值得跟踪的方向。

## 风格建议

优先使用以下视觉特征：
- 宽幅首屏
- 强标题
- 清晰分隔线
- 少量发光或渐变点缀
- 高对比但不刺眼
- 不依赖大量阴影和圆角卡片

## CSS 建议

建议先定义这些变量：
- `--bg`
- `--panel`
- `--text`
- `--muted`
- `--accent`
- `--line`

排版建议：
- 标题使用更有辨识度的衬线或科技感字体组合
- 正文保持克制，行宽不要过长
- 桌面端使用 12 列或 2-3 区分栏
- 移动端退化成单列

## 交互建议

只做轻交互：
- hover 高亮链接
- 时间线条目轻微上浮
- 平台筛选或折叠

不要引入复杂图表或过度动画，除非用户明确要求。
